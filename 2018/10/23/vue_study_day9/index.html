重温vue.js文档——props

prop的大小写

  • HTML中的特性名对大小写不敏感，浏览器编译时会把大写字符解释为小写字母，在使用Dom中的模板时，camelCade的prop名需要使用kebab-case命名,但是如果使用字符串模板，就不会有这个限制（有空去查下原因）

prop类型

  • 列出prop属性的名称和prop各自的名称和类型

传入静态或者动态prop

  • 静态prop

  • 动态

传入数字

  • 说明：传入数字依然需要用v-bind，不然会将数字转化成字符串

传入布尔值

  • 传入布尔值依然需要用v-bind，不然会将数字转化成字符串

传入一个对象

  • 传入对象依然需要用v-bind，不然会将数字转化成字符串

传入一个对象的所有属性

  • 如果你想要将一个对象的所有属性都作为 prop 传入，你可以使用不带参数的 v-bind (取代 v-bind:prop-name)

单向数据流

  • 父级 prop 的更新会向下流动到子组件中，但是反过来则不行。这样会防止从子组件意外改变父级组件的状态，从而导致你的应用的数据流向难以理解。另外，每次父级组件发生更新时，子组件中所有的 prop 都将会刷新为最新的值。（不因该在子组件内部改变prop,如果这样，Vue会在浏览器控制台中发出警告）

    • prop用来传递初始值

    • prop以一种原始的值传入且需要进行转换

      注意在 JavaScript 中对象和数组是通过引用传入的，所以对于一个数组或对象类型的 prop 来说，在子组件中改变这个对象或数组本身将会影响到父组件的状态。

    prop验证

    • 注意： prop 会在一个组件实例创建之前进行验证，所以实例的属性 (如 data、computed 等) 在 default 或 validator 函数中是不可用的。

替换/合并已有的特性

  • class 和 style 会将两边的值会被合并起来 ，而type="text" 就会替换掉 原有的type="date"

禁用特性继承

  • 如果你不希望组件的根元素继承特性，你可以在组件的选项中设置 inheritAttrs: false

  • 通过配合实例$attrs属性使用，该属性包含了传递给一个组件的特性名和特性值。通过inheritAttrs和$attrs，我们可以手动决定这些特性会被赋予那个元素。（撰写基础组件的时候常会用到）

    注意 inheritAttrs: false 选项不会影响 style 和 class的绑定。
